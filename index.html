为什么把析构函数定义为虚函数？

发表于 2016-03-23   |   分类于 C++

先来看一个例子：

会输出什么呢？再改一下。

答案是第一段程序输出

~derived
~base
第二段程序则输出为
~base

执行析构函数时先调用派生类的析构函数，然后才调用基类的析构函数。如果析构不是虚函数，而程序执行时又要通过基类的指针去销毁派生类的动态对象，那么用delete销毁对象时，
只调用了基类的析构，未调用派生类的析构，这样会造成销毁对象不完全。
如果不需要基类对派生类及对象进行操作的时候，则不必定义虚函数，因为定义了虚函数后编译器会给类添加一个虚函数表存放虚函数指针，增加了存储空间。

参见Effective C++

常见C++面试题

发表于 2016-03-23   |   分类于 C++

1.C++多态性有哪些？
静态：编译时刻多态，通过重载函数，类模板多态实现
动态：运行时刻多态，通过虚函数类继承实现
2.C++中类型转换有哪些？
static_cast :最常用的类型转换，如把int转换为float
const_cast :用于取出const属性，例如把const指针转换为非const指针
dynamic_cast :只在转换一个具有至少一个虚函数的类时合法
reinterpret_cast :重新解释数据的二进制，很少用
3.什么是浅拷贝深拷贝？
浅拷贝：用一个现成的对象创建另一个对象时，只拷贝了成员，没有拷贝分配给现成对象中成员的资源
深拷贝：定义对象的拷贝函数，使之资源也拷贝
