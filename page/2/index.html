STL源码

alloc空间配置器

stl_construct.h         定义了全局函数construct()和destroy()，负责对象的构造与析构
stl_alloc.h             定义了一、二级配置器alloc
stl_uninitialized.h     定义了一些全局函数，用来copy或fill大块内存uninitialized_copy/uninitialized_fill等

第一级配置器： 直接使用malloc()和free()

第二级配置器：

  1. 大于128bytes调用第一级配置器
  2. 小于128bytes则检查对应的自由链表free list（8，16，24…128)，如果链表中有可用块直接用
  3. 如果链表中无可用块，调用refill()从内存池中取空间
  4. 如果内存池中空间无法满足需求，则调用malloc为内存池扩充

uninitialized_copy/uninitialized_fill 中调用copy constructor

迭代器与traits

思路：

迭代器是数据容器与算法之间的粘和剂，切合STL的中心思想：将容器与算法分开

设计迭代器无可避免的需要使用型别，如何得到这个型别？ 利用模板的参数推导机制

模板参数推导机制只能推导参数，如果返回值需要用到这个型别则需要声明内嵌型别,函数返回值为typename I::value_type，value_type即声明的内嵌型别

但是，并不是所有的型别都是class type，指针就不是，因此需要偏特化，为指针设计特化版本

综上可引出traits，即特性萃取机，提取内型型别以及提代偏特化(能通过typename *推导出typename)

迭代器型别：

  1. input iterator 最一般
  2. output iterator
  3. forward iterator
  4. bidirectional iterator
  5. random access iterator 最特殊

前三种去持operator++，第四种再加上operator–，第五种涵盖所有指针算术能力p+n,p-n,p[n]等

规定任何迭代器都应提代五个内嵌型别，以便于类型萃取：

  1. iterator_categor ： 迭代器类型
  2. value_type ： 数据类型
  3. difference_type ： 迭代器间距离
  4. pointer ： 数据指针类型
  5. reference ： 数据引用类型

deque

双向开口的连续线性空间(逻辑上看来如此)

与vector的差异：

  • 在于deque允许于常数时间内对起头端进行元素插入或移除操作
  • 是deque没有所谓的“容量”的概念,不会发生“旧空间不足而重新配置一块更大空间”

deque由一段一段的定量连续空间构成，一旦有必要在deque的前端或尾端增加新空间，便配置一段定量连续空间串接在整个deque的头端或尾端。

deque采用一块map(非stl::map)作为主控，map其实是个T**，指向一片连续区域，区域中的值指向一段连续的线性空间称为缓冲区。(问题：map有可能重新整治)

heap

binary heap就是一种complete binary tree完全二叉树，整棵binary tree除了最底层的叶节点之外是填满的，最底层的叶结点从左至右不得有空隙。

以最大堆为例：

push_heap O(1)
新结点是否适用于现在位置？执行上溯程序。
将新节点与父结点比较，如果键值比父节点大，就父子交换，如此一直上溯直到不产生交换

pop_heap O(logN)
执行下溯程序
取出根结点的最大值后，割舍最下层最右边结点，从根结点开始下放：将当前节点与较大子节点对调并持续下放，直到叶点节为止。

sort_heap

最小堆 -> 递减序列

hashtable

如何避免使用一个太大的array来获取常数时间的基本操作？？使用某种映射函数，将大数映射为小数。
可能有不同的元素被映射到相同位置，所以需要解决碰撞问题：线性探测、二次探测

二次探测：如果hash function计算出新元素位置为H，依次尝试H+1,H+2^2,H+3^2,H+4^2….
不是线性探测的H+1,H+2,H+3…..

结论： 如果表格大小为质数，负载系数(元素个数/表格大小)在0.5以下，每插入一个新元素所需要探测次数不多于2

开链： hash function为我们分配一个list，然后在list身上进行元素插入、搜寻、删除等操作。
虽然针对list的搜寻只是一种线性操作，但是list如果够短，速度仍很快(SGI STL 采用开链)

hash_set
运用set目的是快速搜寻元素，set的元素有自动排序功能而hash_set没有

map 与 hash_map 与 unordered_map

c++11增加了unordered系列的容器(unordered_set、unordered_multiset、unordered_map、unordered_multimap），其原理是用hash做的，自然查找速度比原先的map和set要快很多。
其实在C++11标准出来之前，基于hash的容器也已经有了，就是hash_map（在std::tr1内），只是正式标准出来之后，建议统一使用unordered系列的容器。

map的key需要定义operator< ,而unordered_map需要定义hash_value函数并且重载operator==。
对于内置类型，如string，这些都不用操心

类的成员函数指针

C++中，成员指针是最为复杂的语法结构。但在事件驱动和多线程应用中被广泛用于调用回叫函数。在多线程应用中，每个线程都通过指向成员函数的指针来调用该函数。在这样的应用中，如果不用成员指针，编程是非常困难的

一个指向外部函数的指针声明为： 

void (*pf)(char *, const char *); 

void strcpy(char * dest, const char * source); 

pf=strcpy;

一个指向类A成员函数的指针声明为： 

void (A::*pmf)(char *, const char *); 

pmf = &A::strcpy;      //标准C++强制要求加上&号

或者

typedef void(A::*PMA)(char *, const char *); 

PMA pmf= &A::strcat; // pmf是PMF类型(类A成员指针)的变量

通过类成员函数指针调用成员函数

A a;

A *p= &a; //创建指向A的指针

(a.*pmf)(str, “abc”); //将成员函数绑定到pmf

(p->*pmf)(str, “abc”);

LINPHONE

linphone是一款开源的sip电话，支持iOS,Android,Windows Phone8,GNU/Linux，Windows Desktop，MAC OSX以及web浏览器。

linphone内部核心引擎部分和用户接口是分开的。

  • 用户接口前端

    • 给windows和linux提供的GTK+接口
    • 控制台接口
    • Android的JAVA接口
    • IOS的Objective C
    • Windows Phone 的C#接口

  • Liblinphone最重要的核心引擎部分，是实现linphone所有功能的SIP VoIP SDK。

  • Liblinphone依赖于下面的软件组成

    • Mediastreamer2 ： 音视频流创建与处理的多媒体SDK
    • oRTP： RTP库
    • belle-sip SIP库

术语解释

SIP

SIP(Session Initiation Protocol)会话初始协议是一个应用层的信令控制协议。用于创建、修改和释放一个或多个参与者的会话。这些会话可以是Internet多媒体会议、IP电话或多媒体分发。会话的参与者可以通过组播（multicast）、网状单播（unicast）或两者的混合体进行通信。

SIP 使用用户数据报协议(UDP) 以及传输控制协议(TCP)，将独立于底层基础设施的用户灵活地连接起来。

RTP

实时传输协议RTP（Real-time Transport Protocol）是一个网络传输协议。RTP协议和RTP控制协议RTCP一起使用，而且它是建立在用户数据报协议上的。RTP用来为IP网上的语音、图像、传真等多种需要实时传输的多媒体数据提供端到端的实时传输服务。RTP为Internet上端到端的实时传输提供时间信息和流同步，但并不保证服务质量，服务质量由RTCP来提供。

Git

Git是一款免费、开源的分布式版本控制系统。

创建版本库

建一个版本库非常简单，首先，选择一个合适的地方，创建一个空目录

$ mkdir learngit
$ cd learngit
$ pwd
/Users/michael/learngit

第二步，通过git init命令把这个目录变成Git可以管理的仓库：

$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/

将文件添加到版本库

将文件放到learngit目录下（子目录也行），因为这是一个Git仓库，放到其他地方Git再厉害也找不到这个文件

$ git add readme.txt

第二步，用命令git commit告诉Git，把文件提交到仓库：

$ git commit -m "wrote a readme file"
[master (root-commit) cb926e7] wrote a readme file
 1 file changed, 2 insertions(+)
 create mode 100644 readme.txt

-m后面输入的是本次提交的说明，可以输入任意内容，当然最好是有意义的，这样你就能从历史记录里方便地找到改动记录

查看区别

git status命令可以让我们时刻掌握仓库当前的状态

git diff顾名思义就是查看difference，显示的格式正是Unix通用的diff格式

版本控制系统肯定有某个命令可以告诉我们历史记录，在Git中，我们用git log命令查看

用HEAD表示当前版本，上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

$ git reset --hard HEAD^
HEAD is now at ea34578 add distributed

现在总结一下：

HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。

穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本

socket函数

为了执行网络I/O，一个进程必须做的第一件事情就是调用socket函数，指定期望的通信协议类型。

family参数指名协议族或叫做协议域，值可以为：
AF_INET IPV4协议
AF_INET6 IPV6协议
AF_ROUTE 路由套接字

type参数指名套接字类型，值可以为：
SOCK_STREAM 字节流套接字
SOCK_DGRAM 数据报套接字
SOCK_SEQPACKET 有序分组套接字
SOCK_RAW 原始套接字

protocol参数设置为0选择给定的family和type组合的系统默认值。

connect函数

sockfd是由socket函数返回的套接字描述符，第二个与第三个参数分别是一个指向套接字地址接构的指针与该结构的大小。
客户在调用connect前不必非得调用bind函数，因为如果需要的话，内核会确定源ip地址，并选定一个临时端口作为源端口。
调用connect将激发TCP三次握手过程。

bind函数

bind函数把一个本地协议地址赋予一个套接字。

第二个参数是一个指向特定于协议的地址结构的指针，第三个参数是该地址结构的长度。对于TCP，调用bind函数可以指定一个端口号，
或指定一个IP地址，也可以两者都指定。

listen函数

listen函数仅由TCP服务器调用，它做两件事情：
1)listen函数把一个未连接的套接字转换成一个被动套接字，指示内核应接受指向该套接字的连接请求。调用listen导致套接字从CLOSED状态转换到LISTEN状态。
2)规定了内核应该为相应套接字排队的最大连接个数。

内核为任何一个给定的监听套接字维护两个队列：
(1)未完成连接队列：已由某个客户发出并到达服务器，而服务器正在等待完成相应的TCP三次握手过程，套接字处于SYN_RCVD状态。
(2)已完成连接队列：每个已完成TCP三次握手的客户对应其中一项，这些套接字处于ESTABLISHED状态
两队列之和不超过backlog。

accept函数

accept函数由TCP服务器调用，用于从已完成连接队列队头返回下一个已完成连接，如果已完成连接队列为空，那么进程被投入睡眠。

fork与exec函数

fork在子进程返回0而不是父进程的进程ID原因：任何子进程只有一个父进程，而子进程总是可以通过getppid取得父进程的进程ID，
相反，父进程可以有许多子进程，而且无法获取各个子进程的进程ID，如果父进程想跟踪子进程的进程ID，就必须记录每次调用fork的返回值。
fork的两种典型用法:
(1)一个进程创建自身的一个副本，这样每个副本可以在另一个副本执行其他任务同时处理各自操作，这是网络服务器典型用法
(2) 一个进程想要执行另一个程序，于是首先调用fork创建一个自身副本，然后副本调用exec把副本自身替换为新的程序。这是shell之类程序的典型用法。

close函数

用来关闭套接字，并终止TCP连接

getcoskname和getpeername函数

getsockname返回与某个套接字相关联的本地协议地址
getpeername返回与某个套接字相关联的外地协议地址

这两个函数装填由localaddr和peeraddr指向的套接字地址结构。
