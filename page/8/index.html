Ubuntu与WIn7双系统开机找不到Grub修复

发表于 2015-07-18   |   分类于 Linux

有时候装了双系统之后，开机会出现：
“error:file’/boot/grub/i386-pc/normal.mod’not found”
这样的错误，修复方法如下。
1.输入：ls
会显示类似如下信息：
(hd0) (hd0,msdos14) (hd0,msdos13) (hd0,msdos12) …..
2.输入: ls (hd0,msdos14)/grub 有的需要输入：ls (hd0,msdos14)/boot/grub
直到出现的不是error:unknown filesystem为止。
表示找到了grub.cfg文件
3.依次输入：

>set root=(hd0,msdos12)/grub                //假设输入 ls (hd0,msdos12)/grub之后找到cfg文件
>set prefix=(hd0,msdos12)/grub
>insmod normal
>normal

OK了！！

#操作符重载
<<的重载
void operator << (ostream &os,const Time &t)
{
os<<t.hours<<”hours,”<<t.minutes<<”minutes”<<endl;
}
可直接使用cout<<Time_inst;
但是这样定义存在问题：
cout<<”Time is :” <<Time_inst<<”@@\n” ; //不能执行
需要将重载函数改为
ostream & operator << (ostream &os,const Time &t)
{
os<<t.hours<<”hours,”<<t.minutes<<”minutes”<<endl;
return os;
}

1.通过 “#” 符号
module_name (parameter1,parameter2) inst_name (port_map) //inst_name 实例化名字
2.通过 defparam 关键字
与实例化分开，参数需要写绝对路径来指定。
参数传递时各个参数值的排列次序必须与被调用模块中各个参数的次序保持一致，并且参数值和参数的个数也必须相同.
